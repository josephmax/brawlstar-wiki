# 伤害与生存断点

## 定义

伤害—生存断点（breakpoint）指某个明确攻击包对某个明确生存状态所需命中次数发生离散变化。例如从 4 次降到 3 次，或目标原先不带 Shield gear 能承受第 2 次命中、补丁后必须带满盾才能做到。

断点是数值事实，不等于英雄强度、无条件克制或 BP 推荐。它只有与射程、命中可靠性、攻击形态、构筑占用、地图路线和队友补伤结合后，才可能改变稳定 BP 模型。

## 目标生存状态

当前 Ranked 审计统一使用 Power Level 11。Fandom infobox 与补丁中以 Power Level 1 表示的本体血量/伤害，按每级增加 10% 的规则乘以 `2.0`；Shield gear 的 `+900` 是绝对装备值，不再乘等级系数。

一个可计算目标状态至少包含：

- `base_health`：当前形态本体血量。
- `flat_shield`：当前时点可消耗的固定护盾；满值 Shield gear 按维护者规则为 `900`。
- `damage_reductions`：当前时点同时生效、且规则允许叠加的减伤项。
- `active_when`：形态、构筑、资源条、时间窗或技能状态。

本库当前采用维护者提供的加法叠加语义：

```text
total_damage_reduction = sum(damage_reductions)
effective_health = (base_health + flat_shield) / (1 - total_damage_reduction)
```

例如 5000 HP、900 满盾、20% 与 20% 两个同时生效的不同减伤状态，其审计 EHP 为 `5900 / 0.6 = 9833.33…`。这只是特定时点的条件变体，不能写成英雄常驻血量。

## 攻击包

一个可计算攻击包至少包含：

- `damage_per_hit`：被比较的单个离散命中单位。
- `damage_unit`：单发、单枚、单次爆炸、单 tick、完整 ammo 或已声明的组合。
- `hit_count_rule`：一次攻击动作能稳定命中几个该单位。
- `active_when`：形态、距离、构筑、资源、Super / Hypercharge 等条件。
- `source_conflict_status`：数值或语义是否存在来源冲突。

需要的命中次数按 `ceil(EHP / damage_per_hit)` 的数学关系计算，但输出保留原始分数比较。除非来源明确游戏内部每一步舍入顺序，否则报告必须标注为 breakpoint audit arithmetic，不能声称复现了引擎的逐帧取整。

## 平衡补丁差分

同一补丁的 `before` 与 `after` 状态只比较同构攻击包和同构目标变体。高价值差分包括：

- 攻击者从需要 `n+1` 次命中变为 `n` 次。
- 目标从能承受第 `n` 次命中变为会被击杀，或反向变化。
- 基础状态跨线，而 `shield_gear_full` 没有跨线，说明构筑占用可能从可选变为维持该承伤线的必要条件。
- 专属减伤、护盾或形态变化跨线，但基础状态不变，说明变化只属于条件化对位窗口。

百分比生命伤害、治疗/吸血、复活、随时间衰减护盾、下一击减伤、距离连续缩放、多弹散射、DoT 刷新和召唤物耐久必须使用专门攻击包或保持未计算，不能静默降级成普通单发。

## 维护边界

- 计算输入来自 patch source manifest、当前英雄稳定数值/防御变体和可复核机制来源。
- 全矩阵和差分报告写入 `outputs/balance-breakpoints/`。
- 断点报告不得自动生成 strength tier、稳定 map fit、hard gate、slot eligibility、条件化对位边或 runtime 推荐。
- 只有补充机制、`active_when`、`fails_when`、地图/模式职责和 `bp_use` 后，维护者才可把断点证据提升到英雄页现有 BP 字段。
