# 地图知识分层治理

这页定义地图知识在 BP 知识库中的分层方式，避免把稳定地图结构、赛季轮换和版本英雄强度混在同一个页面里。

## 分层原则

地图知识至少分四层：

```yaml
map_knowledge_layers:
  stable_map_entity:
    path: wiki/entities/maps/<map>.md
    changes_when:
      - 地图本体改版
      - 新来源补充稳定地形信息
      - map_feature 抽象被修正
    contains:
      - mode
      - stable map_profile
      - map_features
      - objective_access
      - hero_model_delta
      - false_positive capabilities

  seasonal_rotation_index:
    path: wiki/syntheses/Ranked-Season-<n>-地图Map-Profile总览.md
    changes_when:
      - Ranked 地图池轮换
      - featured mode 改变
      - 地图加入或移出当前池
    contains:
      - 当前赛季地图列表
      - 指向稳定地图实体页的链接
      - 抓取日期和赛季语境

  patch_meta_overlay:
    path: wiki/syntheses/<version-or-meta>.md
    changes_when:
      - 平衡补丁
      - 新英雄上线
      - 竞技 meta 改变
    contains:
      - 当前版本强势英雄
      - 必须 ban / 必须 answer 的英雄
      - 与地图特征叠加后的版本判断

  brawler_map_fit_overlay:
    path: wiki/entities/brawlers/<brawler>.md
    changes_when:
      - 英雄数值或机制改变
      - build 改动
      - 新地图特征被确认
    contains:
      - 该英雄能激活哪些 map_feature
      - 哪些地图是假阳性适配
      - 该英雄在不同模式 / 地图中的 BP 用途
```

## 更新规则

- 赛季轮换时，只更新 seasonal rotation index，不重写稳定地图实体页。
- 地图本体变化时，更新对应地图实体页，并在所有相关赛季索引中保留时间语境。
- 英雄强势变化时，更新版本 / meta 覆盖层和英雄 map-fit，不把强势英雄写死进地图实体页。
- 地图实体页可以记录“能力类型”的价值，例如 `wall_break`、`water_crossing_with_range`、`bush_reveal`，但不应把“当前版本最强英雄”当作稳定地图事实。
- Fandom `Tips` 中出现的英雄名可以作为 `example_brawlers`，但必须标注为来源示例，不等于当前版本推荐。
